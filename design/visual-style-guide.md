# Visual Style Guide

## 1. 视觉关键词

AI Creative Radar 的视觉方向是：

- 复古互联网窗口 UI
- 黑绿色赛博配色
- 老式浏览器窗口
- 黑客终端感
- 像素 / 半调网点图像质感
- 马赛克元素
- Y2K / cyber / weird web
- 个人主页 + 情报档案站
- 神秘、地下、视觉实验感

整体不要做成普通科技资讯站，而要像一个创作者私人的 AI 情报终端。

## 2. 色彩方向

主色：

- 黑色：页面背景和主要界面底色。
- 深灰绿：窗口、面板和卡片底色。
- 荧光绿：标题、按钮、状态、焦点和高亮信息。
- 灰绿色：辅助文字、边框、分割线。

可少量使用：

- 警示黄：重点 Signal 或状态提示。
- 暗红 / 品红：错误、危险、争议、实验标签。

避免：

- 普通白底科技风。
- 蓝紫渐变 SaaS 风。
- 大面积商务蓝。
- 过于干净、没有个性的 AI 工具站风格。

## 3. 字体建议

界面字体可以偏等宽和系统感：

- 标题：等宽字体、像素感字体或压缩型无衬线字体。
- 正文：优先保证可读性，可以使用系统无衬线。
- 标签、按钮、日期、工具名：适合使用等宽字体。

文字风格要像档案、信号、终端输出，而不是营销文案。

## 4. 页面结构感

页面可以像一个旧浏览器或旧系统桌面：

- 顶部是旧浏览器导航栏。
- 主内容像打开的情报窗口。
- 卡片像小窗口、文件夹、档案卡或终端输出块。
- About 页面像一份 `about.txt` 或个人主页说明。

## 5. 顶部导航

顶部导航栏建议：

- 模拟旧浏览器窗口。
- 左侧有窗口控制点或像素按钮。
- 中间可放类似地址栏的区域，例如 `creative-radar://today/signals`。
- 右侧放 Home、Archive、About 入口。

视觉细节：

- 1px 像素边框。
- 直角或轻微圆角。
- 荧光绿描边或聚焦状态。
- 悬停时像旧系统按钮被按下。

## 6. Signal Card

卡片像小窗口或情报档案卡。

建议结构：

- 顶部是窗口标题栏，显示分类和编号。
- 主体显示标题和 signal。
- 底部显示 target_reader、tools 和 visual_tag。
- 按钮使用旧系统按钮风格。

状态设计：

- hover 时边框变成荧光绿。
- 可以增加轻微扫描线或像素噪声。
- 分类标签像终端标签或文件夹标签。

## 7. Today's Signal

Today's Signal 是首页重点区域。

建议视觉：

- 像一条被截获的信号广播。
- 使用更强的荧光绿边框。
- 可加入 `SIGNAL LOCKED`、`TODAY'S SIGNAL`、`RADAR ACTIVE` 等小型状态文字。
- 不要做成普通营销 Hero，要直接展示情报内容。

## 8. 图像和纹理

如果后续加入图片，建议使用：

- 半调网点。
- 扫描线。
- 像素化边缘。
- 低清浏览器截图感。
- 绿色终端色调叠加。
- 马赛克块遮罩。

图片区域不追求高清商业大片感，而是像被归档、截获、压缩后的视觉证据。

## 9. 按钮和标签

按钮：

- 像旧系统按钮。
- 边框清楚。
- 文案短，例如 `OPEN SIGNAL`、`VIEW SOURCE`、`BACK`。
- 按下状态可以有内阴影或位置偏移。

标签：

- 分类标签像文件夹名或终端命令。
- visual_tag 可以像 `#cyber`、`#fashion`、`#prompt`。
- tools 可以像工具清单或依赖列表。

## 10. 布局原则

- 信息要清楚，不能为了风格牺牲阅读。
- 保留足够留白，但不要太现代极简。
- 可以使用网格、边框、分割线和编号形成档案感。
- 移动端要保持单列卡片和可横向滚动分类。

## 11. 禁止方向

- 普通白底资讯站。
- 蓝紫渐变 AI SaaS 首页。
- 大块圆角玻璃拟态卡片。
- 过度商务模板化。
- 只有干净卡片、没有个人风格的工具导航站。
