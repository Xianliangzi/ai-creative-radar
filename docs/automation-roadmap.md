# AI Creative Radar 自动化内容系统路线

## 1. 为什么需要自动化 / Why Automation

AI Creative Radar 未来如果持续更新，不能长期依赖作者手动收集、整理和审核每一条 AI 资讯。

更合理的方向是让系统自动完成大部分重复工作：发现来源、抓取内容、提取信息、判断是否适合视觉创作者、生成情报卡片，并给出可信度判断。

作者的角色不应该是每天逐条复制粘贴内容，而是维护内容质量：处理异常、删除不准确内容、修改分类、置顶重要情报，以及补充优质来源。

## 2. 前台用户和后台维护者 / User vs Admin

### 前台用户

前台用户主要使用这些功能：

- 浏览 AI 情报
- 搜索创意方案
- 查看工具、案例、Prompt、商业玩法
- 打开来源链接
- 复制链接和情报数据
- 反馈链接失效、内容不准确或想看的工具

前台不应该暴露复杂的“生成、审核、发布”流程，否则普通用户会误以为自己需要维护网站内容。

### 后台维护者

后台维护者主要处理这些事情：

- 查看 AI 自动抓取和总结的内容
- 修改分类
- 删除不准确内容
- 调整可信度
- 置顶重要内容
- 手动补充优质来源

后台更像一个轻量内容维护台，而不是普通用户首页的一部分。

## 3. 自动化内容流程 / Content Pipeline

未来自动化内容流程可以设计为：

```text
指定来源
→ 自动抓取 AI 工具官网 / 官方 blog / 创意案例 / AI 新闻源
→ 提取标题、正文、图片和链接
→ AI 判断是否与视觉创作者相关
→ AI 总结成情报卡片
→ AI 可信度评估
→ 自动入库 / 待审核 / 丢弃
→ 网站前台展示
```

这个流程的目标不是“抓取越多越好”，而是筛出对视觉创作者、学生、内容运营和个人创作者真正有用的信息。

## 4. AI 可信度评估机制 / Confidence Level

未来可以使用以下可信度等级：

- `HIGH`：官方来源或明确产品来源，可以优先展示
- `MEDIUM`：可信媒体、创作者案例、工具应用转译，可展示但保留来源
- `LOW`：来源不清、信息不完整、AI 推测内容，进入待审核
- `REJECT`：无关内容、广告垃圾、无法验证内容，不展示

可信度不是为了让 AI “完全自动判断真伪”，而是帮助系统决定内容应该直接展示、进入待审核，还是被丢弃。

## 5. 人工入口的作用 / Human Review

未来人工审核不需要覆盖每一条内容，而是重点处理：

- `LOW` 可信度内容
- 用户举报内容
- 链接失效内容
- 分类错误内容
- 需要置顶的重要内容
- 需要删除的不实内容

人工入口的价值是修正系统无法稳定判断的部分，而不是重复做所有机械工作。

## 6. 未来后台功能设想 / Admin Features

未来后台可能包含：

- 内容列表
- 可信度筛选
- 一键删除
- 修改分类
- 修改标题和摘要
- 标记为已审核
- 置顶情报
- 手动新增来源

这些功能适合放在后台维护界面，不适合直接放在普通用户首页。

## 7. 当前阶段为什么不开发 / Not Now

当前 AI Creative Radar 仍处于前端 MVP 阶段，重点是验证前台浏览、搜索、详情查看、复制和反馈体验。

因此当前暂时不开发：

- 真实自动抓取
- 数据库
- 后台登录
- AI 接口
- 定时任务
- 自动入库

MVP 0.6.11 的重点是先明确产品结构：前台服务普通用户，后台维护内容系统。这样可以避免把后台生成、审核、发布功能放到普通用户首页，造成产品理解混乱。
