# AI Creative Radar 项目复盘

英文名：AI Creative Radar  
中文名：AI 视觉创意情报站  
项目类型：个人 AI Web App / 内容产品 MVP / Vibe Coding 实践项目  
线上地址：[https://ai-creative-radar.vercel.app/](https://ai-creative-radar.vercel.app/)

## 项目亮点 / Highlights

- 从一个泛 AI 新闻网站想法，重新定位为面向视觉创作者的 AI 创意情报站。
- 完成从产品定位、PRD、内容结构、视觉风格到前端开发和上线的完整 MVP 流程。
- 使用 Vite + React 搭建网页，基于 JSON 数据展示 10 条 AI 创意情报内容。
- 支持分类筛选、Signal Card、Signal Detail 弹窗、About 页面和移动端适配。
- 建立了适合后续扩展的图片策略：真实原文图、AI 生成图、分类 placeholder 和人工精选图。

## 项目背景 / Background

我在学习 AI、收集 AI 工具和资讯的过程中，发现一个很明显的问题：AI 信息很多，但真正能转化成视觉创作、作品集项目、Prompt 实验或小商业玩法的内容并不好找。

很多 AI 新闻会告诉用户某个模型发布了、某个工具更新了、某家公司又做了什么，但对于视觉设计、数字媒体艺术、影像、时尚、内容创作方向的学生来说，这些信息和实际创作之间还有一层距离。

我更关心的问题是：

- 这个工具可以用来做什么作品？
- 这个案例能不能放进作品集？
- 这个趋势能不能变成一个视觉项目？
- 这个能力能不能做成内容账号、接单服务、副业或小产品？

所以我把原本的 AI Daily News 方向，调整成了 AI Creative Radar：一个更偏视觉创作者视角的 AI 创意情报站。

## 用户痛点 / User Problems

这个项目主要面向中国学生、数字媒体艺术学生、视觉设计师、AI 视觉创作者、内容运营，以及想用 AI 做作品集、小项目、副业或创意实践的人。

我总结出的核心痛点是：

- AI 新闻太泛，很多内容和普通创作者的实际需求距离较远。
- 工具更新太技术化，新手很难判断它能用于什么创作场景。
- 学生知道 AI 很重要，但不知道怎么把它变成作品集项目。
- 视觉创作者需要的不只是资讯，而是更具体的项目启发、视觉参考和方法路径。
- 普通资讯站缺少创作者视角，很少解释“这条信息可以怎么用”。

## 产品定位 / Product Positioning

AI Creative Radar 不是一个普通 AI 新闻站，而是一个面向视觉创作者的 AI 创意情报站。

它关注的不是单纯的“发生了什么”，而是把 AI 工具、创意案例、视觉趋势和商业玩法，转译成可以行动的 creative signals。

每条内容都尽量回答：

- 这条资讯对视觉创作者有什么用？
- 它可以启发什么项目？
- 它适合谁马上尝试？
- 它有没有作品集、Prompt、内容账号、接单或小产品的可能？

## MVP 功能 / MVP Features

当前 MVP 已经完成以下功能：

- Home 首页：展示产品标题、视觉主图、今日推荐和内容列表。
- Today’s Signal：突出展示当天推荐的一条 AI 创意情报。
- Signal Card：以卡片形式展示每条情报的分类、标题、趋势判断、工具和视觉标签。
- 分类筛选：支持按 AI视觉工具、创意案例、潮流趋势、商业玩法、灵感Prompt、前沿观察筛选内容。
- Signal Detail 弹窗：点击卡片后查看完整内容，包括项目灵感、商业化可能和 Prompt 方向。
- About 页面：说明项目定位、适合人群、作者说明和项目状态。
- 图片 placeholder fallback：没有真实图片时，使用分类化 CSS 视觉占位图。
- 移动端适配：优化手机端 Header、Hero、卡片、详情弹窗和 About 阅读体验。
- Vercel 部署上线：项目已部署为可访问网页。

## 内容结构设计 / Content Structure

我为每条 AI 创意情报设计了结构化 Signal 数据，而不是简单新闻摘要。

核心字段包括：

- `title`：标题，说明这条情报关注的工具、案例、趋势或玩法。
- `signal`：一句话趋势判断，帮助用户快速理解这条内容的价值。
- `summary`：发生了什么，用更容易理解的语言解释背景。
- `creator_value`：对视觉创作者有什么用。
- `project_ideas`：可以做成什么项目，通常给出 2–3 个具体方向。
- `business_potential`：是否适合内容账号、接单、副业、作品集、模板售卖或小产品。
- `target_reader`：适合谁关注。
- `category`：内容分类。
- `tools`：相关工具或平台。
- `prompt_hint`：可执行的视觉提示词方向。
- `source`：来源名称。
- `visual_tag`：视觉风格标签。
- `image` / `image_alt` / `image_mode`：图片展示字段，为后续真实图片、AI 生成图或手动精选图预留。

这个结构的目的，是让内容不只是“信息”，而是可以被创作者继续使用、改造和行动的素材。

## 视觉设计方向 / Visual Direction

视觉风格不是普通 SaaS 风，也不是干净的 AI 工具导航站。我希望它更像一个个人主页式的 AI 视觉档案站。

关键词包括：

- retro browser
- old web
- dark green cyber zine
- visual archive
- halftone
- pixel texture
- 复古电子感
- 个人主页式情报档案站

界面中使用了暗绿色旧电脑 UI、复古浏览器窗口、像素和半调纹理、分类化图片 placeholder，以及类似档案卡的小窗口布局。

这样做的原因是：这个项目本身面向视觉创作者，所以页面不应该只是信息列表，而应该让用户一打开就感受到“视觉情报站”的气质。

## 技术实现 / Tech Stack

项目使用的技术包括：

- Vite + React：搭建前端 MVP。
- JSON 数据驱动：使用 `src/data/news-sample.json` 管理内容。
- 组件化结构：拆分 Header、CategoryFilter、SignalCard、SignalDetail、SignalImage、TodaysSignal、AboutSection 等组件。
- CSS 响应式布局：通过 CSS 完成桌面端和移动端适配。
- GitHub 版本管理：用于保存项目代码和迭代记录。
- Vercel 部署上线：完成线上访问。

当前项目没有接入后端、数据库或 AI 接口，重点是先跑通一个可以被真实访问和展示的 MVP。

## 后续迭代 / Next Steps

后续我希望继续迭代这些方向：

- 替换真实来源链接，让内容更接近真实资讯。
- 接入真实 AI 创意资讯，持续更新内容。
- 半自动生成 Signal，把新闻、工具更新或案例转译成结构化创意情报。
- 自动抓取原文图片，例如 `og:image`、文章封面图或首图。
- 在没有合适原图时，用 AI 生成符合网站视觉风格的配图。
- 建立 Prompt 灵感库。
- 建立 AI 视觉工具库。
- 设计更完整的内容更新流程。
- 继续优化移动端和作品集展示体验。

## 复盘总结 / Reflection

AI Creative Radar 是一个小型个人项目，但它让我更清楚地理解了一个 AI 产品从 0 到 1 的过程。

我从一个宽泛的 AI 新闻想法开始，逐步发现真正想做的不是“更多新闻”，而是“更适合视觉创作者理解和行动的情报”。这个转向也让项目变得更有个人风格。

它目前还不是一个完整成熟的产品，但已经具备了一个作品集项目需要展示的几个部分：明确的问题、清晰的用户、结构化内容、可运行的网页、独立视觉风格和上线结果。

对我来说，这也是一次把 AI 学习、视觉兴趣和网页开发结合起来的实践。
