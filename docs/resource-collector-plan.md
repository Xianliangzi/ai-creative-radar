# AI Creative Radar 资料采集助手规划

## 1. 当前版本定位

当前 `AI 资料采集助手` 是 v1.2 的基础版能力，但它的主要定位已经从“独立资料入库工具”调整为“创意方案的参考资料入口”。

它的目标不是自动抓取全网，也不是直接完成真实性核验，而是先让用户可以把一个链接、工具网站或一段 AIGC 相关文本整理成当前方案可参考的资料卡片。

当前优先级是：

```text
辅助方案生成 > 可选保存进本地资料库
```

也就是说，用户在“创意方案”模块中添加的参考资料，会优先参与：

- 初步方案生成
- AI 方案讨论
- 最终方案整理

如果资料确实有长期价值，用户再选择“保存到本地资料库”。

当前流程：

1. 粘贴链接或文本
2. 系统自动判断输入类型
3. 链接输入会尝试读取网页标题和正文
4. 文本输入会直接交给 AI 整理
5. AI 生成资料卡片预览
6. 资料先进入当前方案的参考资料列表
7. 用户可以选择移除本次参考，或保存到本地资料库

本地新增资料只保存在当前浏览器中，不会写入 `news-sample.json`，也不会同步到云端。

## 2. 参考资料区的产品逻辑

在“创意方案”模块中，参考资料区用于解决一个更真实的使用场景：

```text
用户有一个创意方向
→ 手上有工具网站、文章链接、案例页面或文字资料
→ 希望 AI 参考这些资料生成更贴近需求的方案
→ 有价值的资料再保存进本地资料库
```

参考资料卡片会包含：

- 标题
- 类型 / 分类
- 来源状态
- 核验状态
- 资料意图 `resource_intent`
- 用于方案的提示 `use_in_plan_hint`
- 相关工具
- 相关概念
- 创作者价值
- 可转化项目方向
- Prompt / workflow 建议

其中 `resource_intent` 用于判断资料更像什么：

- `article`：文章
- `tool_site`：工具网站
- `case_page`：案例页
- `github_project`：GitHub 项目
- `social_post`：社交内容
- `unknown`：暂时无法判断

`use_in_plan_hint` 用于说明这条资料可以如何进入方案，例如：

- 工具链参考
- 视觉风格参考
- 案例参考
- 趋势背景参考
- Prompt / workflow 参考

工具官网、产品页、GitHub 项目页不应该被强行当成文章总结。第一版可以根据链接域名、页面标题和可读取文本做保守整理，但必须保留 `verification_status: "待核验"`，不要把未核验信息写成确定事实。

## 3. 当前限制

当前版本只做内容整理，暂不保证以下能力：

- 不保证所有网页链接都能读取成功
- 不做全网相似内容搜索
- 不做真实来源反查
- 不做版权判断
- 不做自动入库到云端数据库
- 不做后台审核系统

因此新增资料会标记：

- `source_status`：链接导入 / 文本导入 / 链接读取失败
- `verification_status`：待核验
- `verification_note`：第一版仅做内容整理，真实性核验将在后续版本增强。
- `local_only`：true

## 4. 后续增强能力

### 4.1 链接读取增强

后续可以优化网页读取能力：

- 更稳定地提取网页标题
- 更准确地提取正文
- 尝试识别发布时间
- 清洗广告、导航、页脚和无关文本
- 对不同站点建立更稳定的解析规则

### 4.2 工具网站识别增强

后续可以增强工具网站和项目页识别：

- 识别工具官网、产品页、GitHub 项目页和文档页
- 判断资料更适合做工具链参考、workflow 参考还是案例参考
- 在网页内容较少时，结合域名、标题和资料库已有知识做更稳妥的保守摘要
- 避免把工具官网误写成新闻或案例文章

### 4.3 文本相似来源匹配

对于用户粘贴的一段文字，后续可以增加来源匹配：

- 从文本中提取关键词
- 搜索网上相似内容
- 找到可能来源链接
- 比对标题、摘要和正文片段
- 给出可能来源和匹配置信度

这一步需要搜索 API 或指定信息源，不应在当前基础版中实现。

### 4.4 真实性 / 可信度核验

后续可以加入资料可信度判断：

- 官方来源优先
- 官方博客、官方文档和产品更新日志优先
- 权威媒体和高质量创作者案例次之
- 搬运号、无来源截图、夸张标题内容降权
- 无法验证来源的内容保持 `待核验`

可信度可以和现有评分字段结合：

- `credibility_score`
- `relevance_score`
- `creator_value_score`
- `actionability_score`

### 4.5 自动抓取与更新

更后期可以考虑自动化内容更新系统：

- RSS 抓取
- 指定网站抓取
- 搜索 API
- 定时任务
- AI 摘要与去重
- 自动评分
- 人工审核后入库

这属于后续自动化内容管线，不属于当前 v1.2 基础采集助手。

## 5. 推荐迭代顺序

1. 先稳定“参考资料辅助方案生成”的完整流程
2. 再优化链接读取质量
3. 再增强工具网站识别
4. 再增加来源核验和相似来源匹配
5. 最后再考虑数据库、后台审核和自动抓取

这样可以避免项目过早变成复杂的爬虫、数据库和内容审核系统。
