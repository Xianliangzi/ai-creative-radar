# AI Creative Radar 资料采集助手规划

## 1. 当前版本定位

当前 `AI 资料采集助手` 是 v1.2 的基础版能力。

它的目标不是自动抓取全网，也不是直接完成真实性核验，而是先让作者或测试用户可以把一个链接或一段 AIGC 相关文本整理成资料库卡片。

当前流程：

1. 粘贴链接或文本
2. 系统自动判断输入类型
3. 链接输入会尝试读取网页标题和正文
4. 文本输入会直接交给 AI 整理
5. AI 生成资料卡片预览
6. 用户确认后保存到本地资料库

本地新增资料只保存在当前浏览器中，不会写入 `news-sample.json`，也不会同步到云端。

## 2. 当前限制

当前版本只做内容整理，暂不保证以下能力：

- 不保证所有网页链接都能读取成功
- 不做全网相似内容搜索
- 不做真实来源反查
- 不做版权判断
- 不做自动入库到云端数据库
- 不做后台审核系统

因此新增资料会标记：

- `source_status`：链接导入 / 文本导入 / 链接读取失败
- `verification_status`：待核验
- `verification_note`：第一版仅做内容整理，真实性核验将在后续版本增强。
- `local_only`：true

## 3. 后续增强能力

### 3.1 链接读取增强

后续可以优化网页读取能力：

- 更稳定地提取网页标题
- 更准确地提取正文
- 尝试识别发布时间
- 清洗广告、导航、页脚和无关文本
- 对不同站点建立更稳定的解析规则

### 3.2 文本相似来源匹配

对于用户粘贴的一段文字，后续可以增加来源匹配：

- 从文本中提取关键词
- 搜索网上相似内容
- 找到可能来源链接
- 比对标题、摘要和正文片段
- 给出可能来源和匹配置信度

这一步需要搜索 API 或指定信息源，不应在当前基础版中实现。

### 3.3 真实性 / 可信度核验

后续可以加入资料可信度判断：

- 官方来源优先
- 官方博客、官方文档和产品更新日志优先
- 权威媒体和高质量创作者案例次之
- 搬运号、无来源截图、夸张标题内容降权
- 无法验证来源的内容保持 `待核验`

可信度可以和现有评分字段结合：

- `credibility_score`
- `relevance_score`
- `creator_value_score`
- `actionability_score`

### 3.4 自动抓取与更新

更后期可以考虑自动化内容更新系统：

- RSS 抓取
- 指定网站抓取
- 搜索 API
- 定时任务
- AI 摘要与去重
- 自动评分
- 人工审核后入库

这属于后续自动化内容管线，不属于当前 v1.2 基础采集助手。

## 4. 推荐迭代顺序

1. 先稳定本地资料采集和资料卡片结构
2. 再优化链接读取质量
3. 再增加来源核验和相似来源匹配
4. 最后再考虑数据库、后台审核和自动抓取

这样可以避免项目过早变成复杂的爬虫、数据库和内容审核系统。
