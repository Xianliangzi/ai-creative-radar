# AI Creative Radar Beta v1.0 版本说明

## 1. 产品定位

AI Creative Radar 是一个面向 AIGC / AI 视觉创作者的灵感雷达与方案脑暴工具。它帮助用户扫描 AI 创意资讯、寻找工具与案例，并把模糊的创意方向整理成可执行的视觉项目方案。

它不是直接生成图片、海报或视频的工具。它更偏向创作前期的资讯收集、灵感整理、方案制定和执行决策，帮助用户想清楚：做什么、怎么做、用什么工具、适合发到哪里，以及如何整理成作品集项目。

当前主要面向：

- AIGC 初学者
- AI 视觉创作者
- 数字媒体艺术学生
- 视觉设计、影像、时尚和内容方向创作者
- 内容运营 / 新媒体运营
- 想做 AI 创意项目、作品集、小红书账号或副业尝试的人

## 2. 当前核心功能

### AI 资讯

- 浏览 AI 工具、创意案例、趋势观察和商业玩法
- 按分类筛选资讯
- 查看情报详情
- 打开原始来源链接
- 复制来源链接
- 复制完整情报数据

### 创意方案

- 输入一个创意方向或自然语言想法
- AI 生成初步方案
- 继续追问，完善方案细节
- 生成完整方案草稿
- 复制完整方案
- 下载 Markdown 文档
- 保存到我的方案库

### 我的方案

- 使用 localStorage 在当前浏览器中本地保存方案
- 查看历史方案
- 复制方案内容
- 下载 Markdown 文档
- 删除方案
- 空状态引导用户去生成一个方案

### 全局搜索

- 顶部搜索框支持真实输入
- 可搜索 AI 资讯
- 可搜索本地保存的我的方案
- 搜索结果区分 AI 资讯和我的方案
- 点击结果可跳转到对应内容
- 支持无结果提示
- 支持清除搜索

## 3. 当前使用流程

1. 用户进入网站。
2. 通过 Start Here 理解三个入口：看 AI 资讯、做创意方案、查看我的方案。
3. 用户浏览 AI 资讯，寻找工具、案例、趋势和商业玩法灵感。
4. 用户进入创意方案。
5. 用户输入一个模糊创意方向，例如数字人作品集、小红书 AI 账号、AI 海报实验或 AI 视频短片。
6. AI 根据用户输入和已有资讯库生成初步方案。
7. 用户继续追问，让 AI 细化执行步骤、视觉风格、平台建议或作品集呈现方式。
8. AI 将初步方案和追问记录整理成完整方案草稿。
9. 用户可以复制、下载 Markdown 文档，或保存到我的方案库。
10. 用户在我的方案中查看历史方案，并进行复制、下载或删除。
11. 用户可以通过顶部搜索快速查找 AI 资讯或已保存方案。

## 4. 已完成的体验优化

- 中文优先 UI 文案，英文作为辅助标签保留。
- 优化产品定位文案，强调这是灵感雷达与方案脑暴工具。
- 明确说明本产品不是直接生成图片、海报或视频的工具。
- 桌面端调整为更宽的工作台布局，减少窄条感。
- 移动端做阅读优先优化，降低装饰线、边框和视觉噪音。
- 首页引导层级简化，减少重复说明模块。
- Start Here 统一入口区用于承接主要使用路径。
- 今日推荐情报归入 AI 资讯 Tab，避免首页层级过多。
- 保存完整方案后，可直接跳转到我的方案。
- 顶部全局搜索从装饰性地址栏变成真实搜索功能。
- 搜索结果支持数量统计、分组展示、无结果提示和清除状态。
- Coming Soon 能力明确标注，避免用户误以为已经可用。
- API 口令前台入口已移除，当前默认开放 AI 试用。

## 5. 当前版本限制

当前 Beta v1.0 还没有实现：

- 登录
- 云端保存
- 多设备同步
- 自动抓取 AI 资讯
- Word 导出
- PPT 导出
- 思维导图
- 后台管理
- 直接生成图片 / 海报 / 视频

我的方案目前使用 localStorage 本地保存，只保存在当前浏览器中。如果清除浏览器数据或更换设备，历史方案可能会丢失。

## 6. 技术状态

- 前端：Vite + React
- 部署：Vercel
- AI API：DeepSeek OpenAI-compatible API
- 数据：当前使用本地 `news-sample.json`
- 方案库：localStorage 本地原型
- 搜索：前端本地搜索，不调用 AI API
- 主要 API：
  - `/api/generate-plan`
  - `/api/refine-plan`
  - `/api/finalize-plan`

说明：真实 API Key 不写入代码，也不提交到 GitHub。AI 配置通过环境变量管理。
